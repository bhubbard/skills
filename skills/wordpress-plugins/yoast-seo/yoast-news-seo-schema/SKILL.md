# Yoast News SEO Schema

The Yoast News SEO plugin modifies the standard `Article` schema piece generated by core Yoast SEO and changes it to the more specific `NewsArticle` type when applicable.

## Reference
[Schema Pieces Documentation](https://developer.yoast.com/features/schema/plugins/#yoast-news-seo)

## Modifying the NewsArticle Schema

You can hook into the specific `NewsArticle` piece to add additional properties required by specific news aggregators or paywalls, such as `isAccessibleForFree` or `dateline`.

```php
add_filter( 'wpseo_schema_article', 'modify_news_article_schema' );
function modify_news_article_schema( $data ) {
    // Ensure we are dealing with a NewsArticle
    if ( isset( $data['@type'] ) && $data['@type'] === 'NewsArticle' ) {
        
        // Example: Add paywall structured data
        $is_premium = get_post_meta( get_the_ID(), '_is_premium_content', true );
        
        if ( $is_premium ) {
            $data['isAccessibleForFree'] = 'False';
            $data['hasPart'] = array(
                '@type' => 'WebPageElement',
                'isAccessibleForFree' => 'False',
                'cssSelector' => '.paywall-content'
            );
        } else {
             $data['isAccessibleForFree'] = 'True';
        }
    }
    
    return $data;
}
```

## Best Practices
- Notice that we use the `wpseo_schema_article` filter, NOT `wpseo_schema_newsarticle`. Yoast News SEO intercepts the core `Article` generator to change its type. Filtering the core article hook is the correct way to modify it.
