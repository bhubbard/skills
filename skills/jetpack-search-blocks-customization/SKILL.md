# Jetpack Search Blocks Customization

Jetpack Search (version 7.0+) allows building full search pages from native Gutenberg blocks. Use this skill when users want to create, design, or customize their search pages using the block editor.

## Core Blocks
*   **Search Input Block**: Provides search-as-you-type functionality.
*   **Filters Blocks**: Includes various filter types like checkboxes, dates, and custom taxonomies. Supports different layouts such as sidebar or collapsible-popover.
*   **Results Blocks**: Displays the search results with selectable layouts, sorting options, and load-more functionality.

## Features
*   **Theme Awareness**: Blocks are theme-aware and work seamlessly on both block and classic themes.
*   **Embedded Search Experience**: Render a full-page search inline in the theme at a real, bookmarkable URL with all features (search-as-you-type, filters, sort, load-more) on the same page.
*   **Overlay Experience**: Deliver the same search blocks in a modal that opens over the current page, keeping visitors in context.
*   **Responsive Design**: Ensure filters collapse appropriately on smaller screens (e.g., below 992px) with a docked trigger, matching the Instant Search overlay UX.

## Guidelines
1.  **Layout Selection**: Choose between the Embedded experience or the Overlay experience based on the site's design requirements.
2.  **Filter Placement**: Configure filter blocks in a sidebar or as collapsible popovers depending on the available screen space and desired user experience.
3.  **Result Customization**: Utilize selectable layouts for the Results block to match the site's aesthetic.
