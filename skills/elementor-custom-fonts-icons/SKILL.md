# Elementor Custom Fonts & Icons

This skill focuses on extending Elementor's typography and visual assets by integrating custom fonts and custom icon libraries without relying on third-party plugins.

## Custom Fonts

Elementor allows you to upload and manage custom font families directly through the WordPress dashboard.

### 1. Supported Formats
For maximum cross-browser compatibility, provide the font in multiple formats:
- **WOFF / WOFF2:** Recommended for all modern browsers (smallest file size, best performance).
- **TTF:** Useful for older browsers or fallback.
- **SVG / EOT:** Legacy formats (rarely needed today).

### 2. Font Variations
- Upload different weights (e.g., 300, 400, 700) and styles (normal, italic) under the same Font Family name to ensure they group correctly in the Elementor typography dropdown.
- Assign the correct CSS weight and style during upload to avoid browser faux-bolding or faux-italicizing.

### 3. Typography Global Settings
- Once uploaded, define global typography variables in Elementor's Site Settings using your new custom fonts. This ensures consistency and makes future updates seamless.

## Custom Icons

Expand beyond the default FontAwesome library by importing tailored vector icon sets.

### 1. Creating Custom Icon Sets
- Use tools like **Fontello**, **IcoMoon**, or **Fontastic** to compile custom SVG icons into a single web font package.
- This approach is highly efficient, as you only load the exact icons you need, reducing page bloat compared to loading entire icon libraries.

### 2. Uploading to Elementor
- Upload the `.zip` file generated by Fontello/IcoMoon directly into Elementor > Custom Icons.
- Once uploaded, a new tab will appear in the Elementor Icon Library interface containing your custom set.

### 3. Inline SVGs
- For one-off icons or multi-colored vectors, enable the "Unfiltered File Uploads" (if safe) or use an SVG sanitizer, and upload SVGs directly via the Elementor Icon Widget using the "Upload SVG" option.

## Performance & Best Practices

- **Preloading:** Consider preloading critical custom fonts in the `<head>` of your site to prevent Flash of Unstyled Text (FOUT) or Flash of Invisible Text (FOIT).
- **Font-Display Swap:** Elementor automatically applies `font-display: swap;` for Google Fonts, but you may need to ensure this is configured correctly for custom fonts via CSS or performance plugins to improve Core Web Vitals (LCP/CLS).
- **Icon Subsetting:** Never upload a custom icon font containing hundreds of unused icons. Keep packages lean.
