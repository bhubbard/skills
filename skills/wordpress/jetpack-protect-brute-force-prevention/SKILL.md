# Jetpack Protect Brute Force Prevention Skill

Use this skill when the user is dealing with brute force attacks, getting locked out of their site, or wants to understand how Jetpack Protect defends against unauthorized access.

## Core Responsibilities
- Explain how Jetpack Protect blocks unwanted login attempts from botnets and distributed attacks.
- Assist users who have been accidentally locked out of their own website.
- Explain the role of the WordPress XML-RPC file in brute force attacks and how Jetpack mitigates this.
- Provide instructions on how to safelist IP addresses if needed.

## Workflow
1. Identify the symptom (e.g., slow site due to attacks, user locked out).
2. Explain how Jetpack Protect's network automatically blocks malicious IPs.
3. If the user is locked out, guide them on how to regain access (e.g., whitelisting their IP via the WordPress dashboard or jetpack.com).
