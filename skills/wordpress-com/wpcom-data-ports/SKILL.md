# WordPress Data Ports (Import / Export)

This skill provides advanced developer guidance for handling Data Ports, specifically focusing on Import and Export functionality within WordPress Studio and the broader WordPress.com ecosystem.

## WordPress Studio Import & Export

WordPress Studio allows developers to seamlessly move site backups and data between different environments using the Import/Export tab.

### Supported Import Formats
You can add new local sites or update existing ones in Studio by importing backups in the following formats:
- `.tar.gz` and `.zip` archives.
- `.wpress` files (generated by All-in-One WP Migration).
- `.sql` files for database-only imports.
- `.zip` files from Local and WordPress Playground.

### Studio Exports
When you export a site from WordPress Studio, the resulting package includes:
- Site content (`wp-content` including plugins, themes, and uploads).
- The `wp-config.php` file.
- A `meta.json` file detailing environment configuration (like PHP versioning).
- A `/sql` directory containing the database data.

## Studio Sync

For sites hosted on WordPress.com (Business or Commerce plans) or Pressable, **Studio Sync** serves as a powerful alternative to manual imports and exports.
- **Pull:** Pull production or staging sites down to your local Studio environment.
- **Push:** Push local changes up to the hosted environment.

## Automation

Import and export capabilities are also available via **Studio Code** and the **Studio CLI**. These tools allow for automated, AI-driven, or command-line site management, making it easier to script environment setup and tear-down.

## Reference
Always refer to the official [WordPress.com Developer Docs](https://developer.wordpress.com/docs/) for the latest features related to Studio imports, exports, and syncing.
