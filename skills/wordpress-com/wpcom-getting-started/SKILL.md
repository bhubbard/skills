# Getting Started with WordPress.com Development

Use this skill when the user asks about creating a new WordPress.com site, setting up a local environment, integrating with GitHub, or deploying code to WordPress.com.

## 1. Create a Site
- Ensure you have a WordPress.com account.
- Create a new WordPress.com site with a Creator or Entrepreneur plan to allow custom plugins and themes.

## 2. Set Up Your Local Environment
- WordPress Studio is the recommended local development environment for WordPress.com. 
- You can develop locally, test changes, and sync them seamlessly.

## 3. Set Up GitHub Integration
- GitHub Deployments allow you to sync a GitHub repository to a WordPress.com site.
- Connect your GitHub account via the WordPress.com dashboard (Settings > Hosting > GitHub Deployments).
- Configure the repository and branch to deploy from.

## 4. Develop Locally
- Use WordPress Studio to run your local site.
- Create and edit themes, plugins, and custom code.
- Commit changes to your Git repository.

## 5. Deploy to Production/Staging
- Push commits to the connected GitHub branch.
- WordPress.com automatically deploys the changes to your staging or production site.
- Alternatively, use WP-CLI or Studio's built-in deployment features for manual syncing.
